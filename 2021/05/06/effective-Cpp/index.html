乱七八糟的相关语法

explicit

  • 将函数声明为explicit，阻止隐式类型转换

~

  • Object o1 = o2; // Object(const Object & rhs)
  • Object o1; o1 = o2; // Object& operator= (const Object & rhs)
  • 函数调用pass by value调用copy构造函数

std::for_each examples

class里的const static

const, enum, #define

  • 取const的地址式合法的，取enum的地址不合法
  • 不聪明的编译器可能在“整数型const对象”上设计另外的存储空间，而enum可以避免非必要的内存分配

const

const and pointers:

  • const std::vector::iterator iter = … // iter不能指向其他东西(T* const)
  • std::vector::const_iterator iter = … // iter指向的东西不可以被改动(T* const)

const成员函数：

  • 让该成员函数可作用于const对象上
  • 两个只有constness不用的成员函数可以被重载
    • 例如 class test{ void f() {}; void f() const {}; };
  • bitwise constness/physical constness 与 logical constness
  • mutable
  • 运用const成员函数生成其对应的non-const版本（反向做法不应该出现）

initialization

  • 成员变量的初始化发生在进入构造函数本体前，除了内置类型 会自动调用这些成员的default构造函数。
  • A::A(): a(0) {b=0;} a调用构造函数初始化，b首先调用default构造函数再进行赋值操作。
  • base早于derived被初始化，class成员变量总是以其声明次序被初始化，即使它们在成员初始列中以不同的次序出现。

不同编译单元内定义的non-local static对象

  • 初始化顺序次序无明确定义，有可能在使用时对象并未构造完成。
  • 将每个non-local static放到函数内成为local static
  • A& f() { static A a; return a; }
  • 多线程系统中带有不确定性。可以在单线程启动阶段，手动调用所有的reference-returning函数
